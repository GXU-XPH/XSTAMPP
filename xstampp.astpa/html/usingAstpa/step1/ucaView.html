Control Actions ← Using the Unsafe Control Actions Table → Corresponding Safety Constraints
Control Actions, added in the view labeled " Control Actions", can have multiple Unsafe Control Action(UCA) that can be marked as hazardous by linking it to one or more Hazards.
A Control Action can have UCA's in four(customizable) categorys, which are each represented by one column in the table:
1Not providing causes hazard
2Providing incorrect causes hazard
3Wrong timing or order causes hazard
4Stoped too soon or applied to long causes hazard

This view does contain three major components:
1 Filtering At the top of the page multiple widgets for filtering the view are available
2 Editing Unsafe Control Actions Each Control Action can contain an abitrary number of UCA entries which are represented as a block of three table cells
3 Adding new Unsafe Control Actions Each Control Action row in the Table has four Add cells for which are displayed

Filtering

Filter Category Type Description
Control Action Text Filter The Table can be filtered by hiding all Control Action rows whose respective Control Actions' title does not contain the typed in text snippet
Unsafe Control Action Text Filter The Table can be filtered by hiding all Unsafe Control Action entrys whose respective Unsafe Control Actions' description does not contain the typed in text snippet
UCA ID Filter by choice The user can choose form a drop-down list of all Unsafe Control Action IDs to show and edit only a single UCA entry
Hazard ID Filter by choice The user can choose form a drop-down list of all Hazard IDs to show and edit only UCA entries that are linked to the chosen Hazard
not hazardous Only by Category If this category is chosen than the all UCA entrys that are linked to a Hazard are hidden from the table. in this filter mode no text input or filter choice is required.

Editing Unsafe Control Actions

Setting the Severity

The severity of an UCA can only be defined if the severity analysis has been activated in the Project Settings.
The current severity of an UCA can be seen and edited in the . By clicking on the widget a severity can be chosen form the Severity dialog.
If an UCA is linked to one or more Hazards than its' severity is automatically set to be higher or equal to the highest severity of the linked Hazards.

Editing and Deleting an Unsafe Control Action

In the center part of an UCA entry a description can be added to the UCA or it the UCA can be deleted from the project.
To edit the description simply click on the gray are in the center to open a text editor, all changes applied can also be undone.
To delete an entry click on which will open a pop-up asking to confirm the deletion Note that this can not be undone

Link a Hazard to an Unsafe Control Action

When linked to a Hazard, the corresponding Control Action is marked unsafe and will appear in the Corresponding Safety Constraints Table. is used to link a Control Action to a Hazard. Press and a list of available Hazards will pop up. Link a Hazard by doubleTclicking on it.
